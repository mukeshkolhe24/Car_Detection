📄 Important Files - What They Do

1️⃣ .gitignore - What NOT to Upload to GitHub

📌 What is it? A file that tells Git which files/folders to ignore when uploading to GitHub.
✅ Why is it important? Prevents uploading large files (datasets, trained models, libraries) that:
  • Are too large for GitHub (>100MB)
  • Users should download/train themselves
  • Are temporary/cache files
# What's in .gitignore:
data/           ← Dataset (user downloads from Kaggle)
work_dirs/      ← Trained models (user trains their own)
mmdetection/    ← MMDetection library (user clones)
*.pth           ← PyTorch model files
*.pyc           ← Python cache files
__pycache__/    ← Python cache folder
build/          ← PyInstaller build files
dist/           ← PyInstaller EXE output
💡 User Action: Nothing! This file is included in the repository.

2️⃣ README.md - Project Overview

📌 What is it? The first file users see on GitHub. Contains project description, installation steps, and usage guide.
✅ Why is it important?
  • Shows project badge (Python version, GPU support)
  • Explains what the project does
  • Provides quick installation commands
  • Shows model performance comparison
# README.md includes:
- Project Title & Badges
- Overview of Faster R-CNN vs YOLO
- Results Comparison Table
- Quick Start Commands
- Requirements
- Folder Structure
💡 User Action: Read this first to understand the project!

3️⃣ setup.bat - One-Click Installation Script

📌 What is it? A Windows batch file that automates the entire installation process.
✅ Why is it important?
  • One-click installation - no manual commands!
  • Creates virtual environment automatically
  • Installs PyTorch, MMDetection, and dependencies
  • Saves time and prevents errors
# What setup.bat does automatically:
1. Creates conda environment (mmdet_car)
2. Activates the environment
3. Installs PyTorch with CUDA support
4. Clones MMDetection from GitHub
5. Installs MMDetection
6. Installs all Python dependencies
7. Displays next steps
💡 User Action: Just double-click setup.bat and wait!

4️⃣ requirements.txt - Python Dependencies

📌 What is it? Lists all Python packages needed for the project with exact versions.
✅ Why is it important?
  • Ensures everyone uses the same package versions
  • One-command installation: pip install -r requirements.txt
  • Prevents version conflicts
# requirements.txt includes:

torchvision==0.15.0

tqdm==4.65.0
tabulate==0.9.0
💡 User Action: Run pip install -r requirements.txt (setup.bat does this automatically)

📦 Quick Installation - Choose Your Method

⚡ Method 1: One-Click (Easiest - Recommended)

✅ Just double-click setup.bat and wait!
This will automatically install everything for you.

🖥️ Method 2: Manual Installation (For Learning)

# 1. Create and activate environment
conda create --name mmdet_car python=3.8 -y
conda activate mmdet_car

# 2. Install PyTorch (with GPU)
pip install torch==2.0.0 torchvision==0.15.0 torchaudio==2.0.0 --index-url https://download.pytorch.org/whl/cu118

# 3. Install MMDetection
pip install -U openmim
mim install mmengine
mim install mmcv==2.0.0
git clone https://github.com/open-mmlab/mmdetection.git
cd mmdetection
pip install -v -e .
cd ..

# 4. Install dependencies
pip install -r requirements.txt

📦 Method 3: Using requirements.txt only

📊 Dataset Setup

⚠️ Dataset is NOT included in GitHub!
You must download it manually from Kaggle.
  1. Go to: Kaggle Car Object Detection Dataset
  2. Download and extract to: data/car_dataset/
  3. Your folder should look like:
data/car_dataset/
├── training_images/          ← All training images
├── testing_images/           ← All testing images
└── train_solution_bounding_boxes.csv
  1. Run data preparation:
python tools/fix_yolo_validation.py
✅ This creates data/annotations/ with COCO format JSON files.

🏃‍♂️ Train Models

⏱️ Training Time:
- Faster R-CNN: ~1.5 hours (12 epochs)
- YOLO: ~2 hours (50 epochs)
- NVIDIA GPU with 4GB+ VRAM recommended

Train Faster R-CNN:

python mmdetection/tools/train.py configs/faster_rcnn/faster_rcnn_car.py --work-dir work_dirs/faster_rcnn_car

Train YOLO:

python mmdetection/tools/train.py configs/yolo/yolo_car.py --work-dir work_dirs/yolo_car
✅ Trained models will be saved in work_dirs/ folder.
✅ Logs will be created automatically with current timestamp.

▶ Run Detection

Option 1: GUI Application (Recommended)

python app_gui/car_detection_app.py

Option 2: Test Single Image

python tools/detect_cars_with_frcnn.py

Option 3: Compare Both Models

python tools/detect_with_comparison.py

Option 4: Check Model Accuracy

python tools/To_find_frcnn_accuracy.py
python tools/To_find_yolo_accuracy.py

🔧 Troubleshooting

❌ "No module named 'mmcv._ext'"
Solution: pip uninstall mmcv mmcv-full && mim install mmcv==2.0.0
❌ "CUDA out of memory"
Solution: Edit config file, change batch_size=2 to batch_size=1
❌ "FileNotFoundError" for images
Solution: Run python tools/fix_yolo_validation.py
❌ "git is not recognized"
Solution: Install Git for Windows from git-scm.com
💡 Always remember:
1. Activate environment first: conda activate mmdet_car
2. Check file paths: dir to verify you're in correct folder
3. Read error messages carefully - they tell you what's missing!
